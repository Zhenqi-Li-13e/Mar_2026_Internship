# VS Code + Python/Rust + Conda + Git 环境搭建完整指南

本笔记记录了 Windows 环境下开发环境的完整搭建流程，包含软件安装、配置以及 Git 版本控制的工作流。

## 一、软件与插件配置

### 1.1 VS Code 安装与插件

| 插件名称 | 用途 |
|---------|------|
| Python | Python 语言支持 |
| rust-analyzer | Rust 语言核心支持 |
| CodeLLDB | Rust 调试（若下载慢可手动安装 VSIX 包） |
| Even Better TOML | TOML 文件语法高亮 |

**版本信息：**
- VS Code：1.87.2
- Python：已通过 Conda 管理
- Git：2.43.0

### 1.2 Conda 环境管理

#### 创建虚拟环境
```bash
conda create -n 环境名 python=版本
conda activate 环境名
```

**示例：**
```bash
conda create -n my_project python=3.12
conda activate my_project
```

#### 安装依赖包
```bash
# 使用 conda-forge 源
conda install -c conda-forge 包名

# 或使用 pip
pip install 包名
```

**优势：** 虚拟环境可实现依赖隔离，避免不同项目间的版本冲突。

### 1.3 Git 与 SSH 配置

#### 生成 SSH 密钥
```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```
默认路径：`C:\Users\xxx\.ssh`（需显示隐藏文件可见）

#### 配置到 GitHub
1. 将公钥（`id_ed25519.pub`）内容添加到 GitHub → Settings → SSH Keys
2. 验证连接：`ssh -T git@github.com`

## 二、Git 完整工作流

> **推荐实践：** 所有操作优先在本地完成（笔记编写、提交），再推送到远程仓库。

### 2.1 基础操作

| 命令 | 说明 |
|------|------|
| `git clone <仓库地址>` | 克隆远程仓库到本地 |
| `git status` | 查看文件跟踪状态 |
| `git add .` | 跟踪所有文件（暂存） |
| `git commit -m "备注"` | 提交到本地仓库 |
| `git push -u origin <分支名>` | 推送到远程分支 |

### 2.2 分支管理

```bash
# 创建并切换新分支
git checkout -b feature/功能名

# 切换分支
git checkout main

# 合并功能分支到主分支
git merge feature/功能名
```

### 2.3 分支清理

```bash
# 删除本地分支
git branch -d feature/功能名

# 删除远程分支
git push origin --delete feature/功能名
```

### 2.4 完整示例（以本仓库为例）

```bash
# 1. 克隆仓库到 D 盘（避免 C 盘占用）
git clone git@github.com:username/Omaka-Internship.git D:\GitHub\Omaka-Internship

# 2. 创建功能分支
git checkout -b feature/env_note

# 3. 编写笔记并提交
git add .
git commit -m "添加环境搭建笔记"

# 4. 推送到远程分支
git push -u origin feature/env_note

# 5. 合并到主分支
git checkout main
git merge feature/env_note
git push origin main

# 6. 清理分支
git branch -d feature/env_note
git push origin --delete feature/env_note
```

## 三、核心原则

1. **环境隔离**：使用 Conda 虚拟环境隔离不同项目依赖，避免版本冲突
2. **分支开发**：功能分支开发可避免主分支代码混乱
3. **本地优先**：所有修改先在本地完成，确认无误后再推送到远程
4. **仓库位置**：建议将仓库放在 D 盘，避免 C 盘空间占用

> **注意：** 中文文件名在 Git 中可能出现乱码，但不影响实际操作。
